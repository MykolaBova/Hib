Butler

"Table Oriented Programming" -- TOP

Neat and simple.  Not widely used.

Reads JDBC schema, creates a hash map of property/values.  No need for reflection.

No attempt at an ORM -- no identity, no references.  Not even a schema at the Java level.

Has a JSP level query language rather than raw SQL.  I'd prefer that aspect to be kept simple.

Very cute and simple.  Too simple?  The tool looks very productive but will not be considered to be cool.

Single developer Fredrik Bertilsson.  Very quite project.

Very rough documentation.

Note there are two unrelated Java projects named Butler.  This one is at http://www.javaworld.com/javaworld/jw-10-2004/jw-1018-butler.html or http://butler.sourceforge.net/

Trails

http://www.trailsframework.org/
Chris Nelson java.net into article http://today.java.net/pub/a/today/2005/06/23/trails.html and advanced http://today.java.net/pub/a/today/2005/11/04/further-down-the-trail.html

  • Trails  https://trails.dev.java.net/ -- Ruby Rails inspired.
  • Hibernate POJO data bean + Attibs.   Lots of reflection.
  • Auto CRUD from POJO.
  • "DDD"  Domain Driven Development.  Busness rules with objects.  Minimize layers.
  • Uses standard, cool, complex, overengineered technologies: Tapestry, Hibernate.
  • Single developer ccnelson (Chris) but some commutity starting.
  • Lousy docs, but this is recognized.
  • Generally very interesting project.
  • Now Tapestry?
