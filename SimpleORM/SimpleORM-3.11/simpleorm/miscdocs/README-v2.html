SimpleORM README

Introduction

SimpleORM provides a very simple and easy way to map relational tables into java classes. It uses a novel generalized design to avoid the need for reflection and numerous configuration files. SimpleORM takes care with database semantics such as locking and caching, and also alows from raw JDBC and non-Java database access.

The SimpleORM White Paper provides a detailed description of how SimpleORM works. It is strongly recomended that it be read before attempting to run SimpleORM.

This is a new project, but all of the core features work and have been tested. Several databases are supported, including PostgreSQL, MySQL, HSQL, and Oracle. The documentation is good and there is a growing set of examples.

Because it is simple, very few bugs have been reported, and reported bugs have been fixed very quickly. The next release or two will be stable, production ready.

As to simplicity, the .jar is currently just 68kb. The code is well documented, and we have taken care with things like toString() and exceptions. So if you do decide to use it seriously it should not be hard to work with the source.

Unpacking and Building

The distribution contains the entire source for SimpleORM under the simpleorm folder. It is slightly unusual in that it combines java package folders at the same level as documentation and build folders -- we like broad shallow hierarchies rather than narrow deep ones. It contains the following folders.

simpleorm/ This contains all only the sources, both java and others. other projects may add an extra level called src, but we prefer to keep things simple.
simpleorm/core The core simpleorm classes themselves. The are in package simpleorm.core.
simpleorm/properties The simpleorm property classes that may be shared with a UI layer in the future.
simpleorm/examples The test and demonstration classes.
simpleorm/build  Build scripts. SimpleORM is built with apache ant, see build.xml for details. This does NOT contain the built .class files.
simpleorm/../built  Everything that is built by simpleorm. The ant build script creates this folder and all subfolders.
simpleorm/../lib  .jar files to add to the class path go here. In particular your JDBC driver. You create this folder.
simpleorm/webSite  This gets posted to the SimpleORM.org website. Includes the white paper.
simpleorm/extras/SimpleORMGenerate
 Generates SimpleORM definitions from existing table DDL using JDBC.
simpleorm/extras/EJB/  Demonstrates how to use simpleORM with EJBs.

Having unziped the distribution you may need to set up a simpleorm.properties file in order to obtain a proper JDBC connection. This is described in build.xml.

ant will do a clean build, (re)create the tables, and run the test program which demonstrates most of SimpleORM's features. ant api will make the documentation.

The Examples

The examples package/directory contains examples and test cases based on a simple schema involving Department/Employee/Payroll tables. Start with ADemo, which is run as the default ant task (for the time being). It demonstrates how to open a connection, and perform DML on records.
